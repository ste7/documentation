---
title: "Update Current User"
openapi: "PUT /v1/users/me"
---

Update information about the currently authenticated user.

<Warning>
  The `username` field must be unique across both users and teams. If you
  attempt to update to a username that already exists, the request will fail.
</Warning>

## Request Body

<ParamField body="username" type="string" required>
  The unique username for the user. Must be unique across users and teams.
  Maximum 255 characters. Must match pattern: `^[a-z0-9][a-z0-9\-\._]+[a-z0-9]$`
  (must start and end with alphanumeric, can contain hyphens, dots, and
  underscores in between).
</ParamField>

<ParamField body="email" type="string" required>
  The user's email address. Must be unique. Maximum 255 characters.
</ParamField>

<ParamField body="first_name" type="string" required>
  The user's first name. Maximum 255 characters.
</ParamField>

<ParamField body="last_name" type="string">
  The user's last name. Maximum 255 characters.
</ParamField>

<ParamField body="avatar" type="file">
  Avatar image file. Must be an image file (jpeg, png, jpg, svg). Maximum size:
  1024 KB.
</ParamField>

<ParamField body="timezone" type="string" required>
  The user's timezone (e.g., "America/New_York"). Must be a valid timezone
  identifier.
</ParamField>

<ParamField body="time_format" type="integer" required>
  The user's preferred time format. Must be either `12` (12-hour format) or `24`
  (24-hour format).
</ParamField>

<ParamField body="start_of_week" type="integer" required>
  The day that starts the week. Must be either `0` (Sunday) or `1` (Monday).
</ParamField>

## Example Request

<RequestExample>
```json
{
  "username": "johndoe",
  "email": "john@example.com",
  "first_name": "John",
  "last_name": "Doe",
  "timezone": "America/New_York",
  "time_format": 12,
  "start_of_week": 1
}
```
</RequestExample>

## Response

<ResponseField name="success" type="boolean" required>
  Indicates whether the request was successful. Always `true` for successful
  responses.
</ResponseField>

<ResponseField name="data" type="object">
  The updated user object.
  
  <Expandable title="User properties">
    <ResponseField name="id" type="integer" required>
      The unique identifier of the user.
    </ResponseField>

    <ResponseField name="first_name" type="string" required>
      The user's first name.
    </ResponseField>

    <ResponseField name="last_name" type="string" required>
      The user's last name.
    </ResponseField>

    <ResponseField name="username" type="string" required>
      The unique username for the user.
    </ResponseField>

    <ResponseField name="email" type="string" required>
      The user's email address.
    </ResponseField>

    <ResponseField name="timezone" type="string" required>
      The user's timezone (e.g., "America/New_York").
    </ResponseField>

    <ResponseField name="time_format" type="string" required>
      The user's preferred time format (e.g., "12" or "24").
    </ResponseField>

    <ResponseField name="start_of_week" type="integer" required>
      The day that starts the week (0 for Sunday, 1 for Monday).
    </ResponseField>

    <ResponseField name="avatar" type="string">
      URL to the user's avatar image. Returns `null` if no avatar is set.
    </ResponseField>

  </Expandable>
</ResponseField>
