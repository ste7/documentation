---
title: "Update Team"
openapi: "PUT /v1/teams/{id}"
---

Update an existing team. You can only update teams where you are the owner.

<Warning>
  If you update the `username` field, it must remain unique across both teams
  and users. Updating to an existing username will cause the request to fail.
</Warning>

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the team to update.
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  The name of the team. Maximum 55 characters.
</ParamField>

<ParamField body="username" type="string" required>
  The unique username for the team. Must be unique across teams and users if
  changed. Maximum 255 characters. Must match pattern: `^[a-zA-Z0-9\-\_]+$`.
</ParamField>

<ParamField body="about" type="string">
  Description or about text for the team. Maximum 5000 characters.
</ParamField>

<Note>
  All request body fields are required for update. The `avatar` field cannot be
  updated through this endpoint.
</Note>

## Example Request

<RequestExample>
```json
{
  "name": "Acme Corporation Inc",
  "username": "acme-corp",
  "about": "Leading provider of innovative solutions worldwide"
}
```
</RequestExample>

## Example Response

<ResponseExample>
```json Success (202 Accepted)
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Acme Corporation Inc",
    "username": "acme-corp",
    "about": "Leading provider of innovative solutions worldwide",
    "logo": "https://example.com/logos/acme.png"
  }
}
```
</ResponseExample>

## Error Response

<ResponseExample>
```json Error (403 Forbidden)
{
  "success": false,
  "error": {
    "code": "ACCESS_DENIED",
    "message": "You are not authorized to update this team."
  }
}
```
</ResponseExample>

<ResponseExample>
```json Error (400 Bad Request)
{
```

</ResponseExample>
